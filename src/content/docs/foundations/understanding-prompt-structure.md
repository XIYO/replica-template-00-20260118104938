## 프롬프트 구조화의 중요성

대규모 언어 모델(LLM)과 효과적으로 소통하기 위해서는 단순히 질문을 던지는 것을 넘어, 모델이 이해할 수 있는 체계적인 구조로 명령을 전달해야 합니다. 잘 구조화된 프롬프트는 모델의 '환각(Hallucination)' 현상을 줄이고, 사용자가 의도한 결과에 더 가까운 답변을 생성하도록 돕는 설계도와 같습니다.

프롬프트의 핵심 구성 요소는 크게 세 가지로 나뉩니다: **지시어**, **컨텍스트**, **입력 데이터**.

---

## 1. 지시어 (Instruction)

지시어는 모델이 수행해야 할 **구체적인 작업**을 정의합니다. 프롬프트에서 가장 핵심적인 부분이며, 명령은 모호하지 않고 직접적이어야 합니다.

- **나쁜 예:** "이 글에 대해 알려줘."
- **좋은 예:** "다음 뉴스 기사의 핵심 내용을 3개의 불렛 포인트로 요약해줘."

### 지시어 작성 팁
- '하지 마라'는 부정문보다 '해라'라는 **긍정문**을 사용했을 때 더 정확한 결과를 얻는 경우가 많습니다.
- "추출해라", "분류해라", "번역해라"와 같은 **명확한 동사**를 사용하세요.

## 2. 컨텍스트 (Context)

컨텍스트는 모델이 작업을 수행하는 데 필요한 **배경 정보나 제약 조건**을 제공합니다. 이는 결과의 톤앤매너와 품질을 결정짓는 중요한 요소입니다.

- **역할 설정(Persona):** "너는 10년 차 전문 마케팅 컨설턴트야."
- **청중 설정:** "기술적 지식이 없는 초보자에게 설명하듯이 작성해줘."
- **제약 사항:** "답변은 200자 이내로 제한하고, 전문 용어 사용을 지양해줘."

컨텍스트가 상세할수록 모델은 불필요한 추측을 줄이고 사용자의 의도에 부합하는 답변을 내놓습니다.

## 3. 입력 데이터 (Input Data)

입력 데이터는 모델이 실제로 **가공하거나 분석해야 할 대상**입니다. 지시어와 컨텍스트가 '어떻게 할 것인가'를 결정한다면, 입력 데이터는 '무엇을 할 것인가'에 해당합니다.

데이터가 길어질 경우, 구분자(Delimiter)를 사용하여 지시 사항과 실제 데이터를 분리하는 것이 좋습니다. 예를 들어 `"""`, `---`, `###` 등을 활용하여 경계를 명확히 합니다.

```text
### 지시어
아래의 고객 피드백 내용을 감성 분석하여 '긍정' 또는 '부정'으로 분류하세요.

### 입력 데이터
"배송이 예상보다 늦었지만, 제품의 품질은 기대 이상으로 훌륭합니다. 재구매 의사 있습니다."
```

## 실전 활용 예시

위의 세 요소를 결합하여 완성도 높은 프롬프트를 구성해 보겠습니다.

```markdown
# 지시어
다음 제품 사양 정보를 바탕으로 인스타그램 홍보 문구 2개를 작성해줘.

# 컨텍스트
- 타겟: 홈 오피스를 꾸미고 싶어 하는 1인 가구 직장인
- 톤앤매너: 감성적이면서도 실용성을 강조하는 말투
- 포함 요소: 이모지를 활용하고, 답변은 한국어로 작성할 것

# 입력 데이터
제품명: 슬림 무소음 기계식 키보드
특징: 저소음 적축, 파스텔 블루 컬러, 블루투스 3대 멀티 페어링 지원
```

이와 같이 구조화된 프롬프트는 모델에게 명확한 가이드라인을 제공하여, 반복적인 수정 과정 없이도 한 번에 고품질의 결과를 얻을 수 있게 해줍니다.
