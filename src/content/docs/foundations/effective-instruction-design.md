## 서론

대규모 언어 모델(LLM)로부터 원하는 결과를 얻기 위한 가장 중요한 첫 단계는 **명확한 지시문(Instruction)**을 설계하는 것입니다. 지시문이 구체적일수록 모델은 수행해야 할 작업의 범위를 정확히 파악하며, 결과물의 품질과 일관성이 비약적으로 향상됩니다. 본 문서에서는 모호함을 제거하고 성공적인 결과를 이끌어내는 핵심 원칙을 설명합니다.

## 1. 명확하고 구체적인 지시

모델은 독심술사가 아닙니다. 사용자의 의도를 추측하게 하지 말고, 구체적인 요구사항을 텍스트로 명시해야 합니다.

### 지시문 개선 사례

- **나쁜 예**: "이 글을 요약해줘."
- **좋은 예**: "다음 뉴스 기사를 3문장 이내의 블렛 포인트 형식으로 요약하세요. 독자는 일반 대중이며, 전문 용어는 쉬운 단어로 풀어서 설명해야 합니다."

## 2. 역할 부여 (Persona)

모델에게 특정한 페르소나를 부여하면 답변의 톤앤매너와 깊이가 달라집니다. 어떤 관점에서 답변해야 하는지 명시하세요.

```text
지시문: "당신은 10년 차 시니어 소프트웨어 엔지니어입니다. 신입 개발자가 작성한 아래의 Java 코드를 코드 리뷰하고, 성능 최적화와 가독성 측면에서 개선점을 제안하세요."
```

## 3. 구분자(Delimiters) 활용

지시 사항과 입력 데이터를 명확하게 구분하면 모델이 혼동할 가능성이 줄어듭니다. `"""`, `###`, `---`, `<tag>` 등의 기호를 적극 활용하세요.

```text
### 지시 사항
아래의 [텍스트] 섹션에 제공된 고객 리뷰를 긍정 또는 부정으로 분류하세요.

### 텍스트
"배송이 예상보다 빨랐고 제품 상태도 훌륭합니다. 다만 가격이 조금 비싼 편이네요."

### 출력 형식
분류: (긍정/부정)
이유: (한 문장으로 설명)
```

## 4. 단계별 작업 정의 (Chain of Thought)

복잡한 작업은 한 번에 요청하기보다 논리적인 단계로 나누어 지시하는 것이 효과적입니다.

1.  **분석**: 먼저 텍스트에서 주요 키워드를 추출하세요.
2.  **분류**: 추출된 키워드를 바탕으로 주제를 카테고리화하세요.
3.  **작성**: 분류된 카테고리를 기반으로 최종 보고서를 작성하세요.

## 5. 부정문보다 긍정문 사용

"~하지 마세요"라는 지시보다 "~하세요"라는 지시가 모델에게 더 명확한 가이드를 제공합니다.

- **비효율적**: "반말을 사용하지 마세요."
- **효율적**: "정중하고 격식 있는 문어체(~하십시오 체)를 사용하여 답변하세요."

## 결론

효과적인 지시문 설계는 한 번에 완성되지 않습니다. 위 원칙들을 적용한 후 모델의 답변을 분석하고, 부족한 부분을 보완하며 지시문을 다듬어 나가는 반복적인 프로세스가 필수적입니다. 명확한 맥락과 제약 조건을 제공할수록 모델은 여러분의 강력한 조력자가 될 것입니다.
