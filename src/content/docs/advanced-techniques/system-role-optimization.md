## 개요

대규모 언어 모델(LLM)과의 상호작용에서 **시스템 역할(System Role)**은 모델의 행동 지침을 설정하는 가장 강력한 도구입니다. 페르소나를 명확히 정의함으로써 AI는 단순한 텍스트 생성기를 넘어, 특정 분야의 전문가나 특정 규칙을 준수하는 가상 조력자로 기능하게 됩니다.

## 페르소나 설정의 핵심 요소

효과적인 시스템 메시지를 작성하기 위해서는 다음 세 가지 요소가 포함되어야 합니다.

1.  **역할 정의 (Identity)**: AI가 '누구'인지 명시합니다. (예: 시니어 보안 엔지니어, 초등학교 교사)
2.  **어조 및 문체 (Tone & Voice)**: 응답의 분위기를 설정합니다. (예: 정중하게, 간결하게, 위트 있게)
3.  **제약 사항 및 규칙 (Constraints)**: 반드시 지켜야 할 규칙이나 금지 사항을 정의합니다.

## 실전 최적화 전략

### 1. 구체적인 상황 설정

단순히 "당신은 전문가입니다"라고 하기보다, 구체적인 경력과 상황을 부여할 때 응답의 깊이가 달라집니다.

```json
{
  "role": "system",
  "content": "당신은 15년 경력의 파이썬 아키텍처 전문가입니다. 코드 리뷰 시 가독성보다 성능 최적화와 메모리 효율성을 최우선으로 고려하여 피드백을 제공하세요."
}
```

### 2. 단계별 사고(Chain of Thought) 유도

시스템 메시지에 사고 프로세스를 정의하면 모델의 논리적 오류를 줄일 수 있습니다.

- "모든 질문에 대해 먼저 문제의 핵심을 파악한 뒤, 해결 방안을 3단계로 나누어 설명하세요."
- "결론을 내기 전에 반드시 반대 사례(Edge case)를 검토하세요."

### 3. 출력 형식 제어

일관된 응답 구조를 유지하기 위해 마크다운 헤더나 JSON 형식을 강제할 수 있습니다.

```markdown
- 모든 응답은 ## 요약, ### 분석, ### 권장 사항의 형식을 따를 것.
- 전문 용어를 사용할 때는 반드시 괄호 안에 쉬운 설명을 덧붙일 것.
```

## 페르소나 적용 예시

### 예시: 시니어 보안 감사관

**System Message:**
> 당신은 금융 시스템 전문 보안 감사관입니다. 제출된 코드에서 OWASP Top 10 취약점을 찾아내고, 보안 등급(Low, Medium, High)을 매겨 리포트 형식으로 출력하세요. 해결책은 반드시 실제 코드 스니펫을 포함해야 합니다.

### 예시: 창의적 글쓰기 코치

**System Message:**
> 당신은 신춘문예 당선작을 다수 배출한 글쓰기 코치입니다. 사용자의 글에서 진부한 표현(Cliché)을 찾아내어 신선한 은유로 바꾸는 제안을 하세요. 어조는 독려하듯이 따뜻해야 합니다.

## 결론

잘 설계된 페르소나는 프롬프트 엔지니어링의 핵심입니다. 시스템 메시지를 통해 AI의 지식 범위와 행동 양식을 정교하게 제어하면, 매번 다른 결과가 나오는 불확실성을 제거하고 비즈니스 요구사항에 부합하는 일관된 품질의 결과를 얻을 수 있습니다.
